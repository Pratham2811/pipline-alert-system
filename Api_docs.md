# PI AI Pipeline Alert System API

This API provides endpoints to manage and monitor alerts generated by the PI AI pipeline. It supports full CRUD operations on alerts as well as summary and statistics endpoints for analytics.

**Base URL:** `http://localhost:5000/api/alerts`

---

## Endpoints Overview

| Method     | Endpoint      | Description           |
| :--------- | :------------ | :-------------------- |
| **GET**    | `/`           | Get all alerts        |
| **GET**    | `/:id`        | Get alert by ID       |
| **POST**   | `/`           | Create a new alert    |
| **PUT**    | `/:id`        | Update an alert       |
| **DELETE** | `/:id`        | Delete an alert       |
| **GET**    | `/summary`    | Get alerts summary    |
| **GET**    | `/statistics` | Get alerts statistics |

---

## Endpoint Details

### 1. Get All Alerts

Retrieves a list of all alerts in the system. Returns an array of alert objects with their details, including detection method, location, and status.

- **Method:** `GET`
- **Endpoint:** `/`

**Response (200 OK)**

```json
{
  "success": true,
  "data": [ ...array of alert objects... ]
}
2. Get Alert by ID
Retrieves a single alert by its unique MongoDB ObjectId.

Method: GET

Endpoint: /:id

Path Parameters

id (string, required): The MongoDB ObjectId of the alert (e.g., 69cf6cdb14f3e69aa3fd28e6).

Response (200 OK)

JSON
{
  "success": true,
  "data": { ...alert object... }
}
3. Create Alert
Creates a new alert in the system.

Method: POST

Endpoint: /

Request Body

detection_method (string, required): The method used to detect the alert (e.g., Method 1, Method 2).

location_km (number, required): The location in kilometers.

status (string, optional): The current status of the alert (e.g., active, resolved).
(Note: Additional fields such as pipeline_segment, alert_type, and severity are supported as shown in the example below).

Example Request Body

JSON
{
  "pipeline_segment": "Tamil-Nadu-Central-09",
  "detection_method": "Method 2",
  "alert_type": "leak",
  "severity": "medium",
  "location_km": 55.0,
  "status": "active"
}
Response (201 Created)

JSON
{
  "success": true,
  "data": { ...newly created alert object... }
}
4. Update Alert
Updates an existing alert by its unique MongoDB ObjectId. Send only the fields you want to update in the JSON body.

Method: PUT

Endpoint: /:id

Path Parameters

id (string, required): The MongoDB ObjectId of the alert to update.

Request Body

detection_method (string, optional): Updated detection method.

location_km (number, optional): Updated location in kilometers.

status (string, optional): Updated status of the alert.

severity (string, optional): Updated severity level.

Example Request Body

JSON
{
  "severity": "low",
  "location_km": 55.2,
  "status": "resolved"
}
Response (200 OK)

JSON
{
  "success": true,
  "data": { ...updated alert object... }
}
5. Delete Alert
Deletes an alert by its unique MongoDB ObjectId.

Method: DELETE

Endpoint: /:id

Path Parameters

id (string, required): The MongoDB ObjectId of the alert to delete (e.g., 69cf7fc593bbf8d399a381b0).

Response (200 OK)

JSON
{
  "success": true,
  "message": "Alert deleted successfully"
}
6. Get Alerts Summary
Returns a high-level summary of all alerts in the system. Includes counts and aggregated statistics grouped by relevant fields such as detection method and status.

Method: GET

Endpoint: /summary

Response (200 OK)

JSON
{
  "success": true,
  "data": { ...summary object... }
}
7. Get Alerts Statistics
Returns detailed statistics about alerts in the system, including total count, breakdown by detection method, and average location for active alerts.

Method: GET

Endpoint: /statistics

Response (200 OK)

JSON
{
  "success": true,
  "data": {
    "total_alerts": 5,
    "alerts_by_detection_method": [
      { "count": 2, "detection_method": "Method 1" },
      { "count": 2, "detection_method": "Method 2" },
      { "count": 1, "detection_method": "Method 3" }
    ],
    "average_location_km_active": 127.65
  }
}
```
